# Contributing Guidelines

Thank you for considering contributing to this project!  

## How to Contribute
1. Fork the repository.
2. Create a new branch for your changes.
3. Make your edits (scripts, datasets, documentation).
4. Run the R check workflow to ensure everything passes.
5. Submit a pull request with a clear description of your changes.

## Standards
- Use clear and consistent naming for files and variables.
- Place datasets in the `datasets/` folder.
- Place case studies in the `case_studies/` folder.
- Include comments in your R scripts for readability.

## Reporting Issues
If you find a bug or have a suggestion, please open an issue in the repository.
